## Quick Tips

1. **Stop after 3 attempts**: Use systematic-debugging skill instead of guessing
2. **Trace backwards**: Follow execution from error to source
3. **Add instrumentation**: When execution path is unclear
4. **Isolate variables**: Change one thing at a time
5. **Document findings**: Track what you've tried
